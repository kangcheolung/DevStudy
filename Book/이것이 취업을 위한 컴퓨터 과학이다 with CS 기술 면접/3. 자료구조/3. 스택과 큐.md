# 스택과 큐

## 스택

스택은 한 쪽에서만 데이터의 삽입 및 삭제가 가능한 자료구조를 말한다. 흔히 스택에 저장된 데이터를 저장하는 연산은 “”푸시””, 반대로 데이터를 빼내는 연산은 “”팝””이라고 한다. 이렇게 한 쪽에서만 데이터가 저장, 관리되면 나중에 삽입된(후입) 데이터가 먼저 나오게(선출) 된다. 이러한 점에서 스택을 “”후입선출(LIFO)””자료구조라고 이야기한다. 

## 큐

큐는 기본적으로 한 쪽으로 데이터를 삽입하고, 다른 한 쪽으로 데이터를 삭제할 수 있는 자료구조이다. 이렇게 데이터가 저장, 관리되면 먼저 삽입된(선입) 데이터가 먼저 나오게(선출) 된다. 이런 점에서 큐를 선입선출(FIFO) 자료구조라고 이야기한다. 

흔히 큐의 한 쪽 끝에 데이터를 삽입하는 연산을 “”인큐””라고 하고, 다른 한 쪽 끝으로 데이터를 빼내는(삭제하는) 연산을 “”디큐””라고 한다. 

큐는 임시 저장된 데이터를 차례차례 내보내거나 꺼내와야 하는 각종 버퍼로도 활용된다. 

### 원형 큐

원형 큐는 데이터를 삽입하는 쪽과 삭제하는 쪽, 양 쪽을 하나로 연결해 원형으로 사용하는 자료구조이다. 

### 덱

덱은 양방향 큐의 약자로, 양쪽으로 데이터를 삽입/삭제할 수 있는 큐를 말하며, 우선순위 큐는 저장된 요소들이 선입선출로 처리되는 것이 아니라 정해진 우선순위가 높은 순으로 처리되는 큐이다. 

우선순위 큐는 합이라는 자료구조를 기반으로 구현되는데, 힙을 이해하려면 트리가 무엇인지 먼저 이해해야한다.
