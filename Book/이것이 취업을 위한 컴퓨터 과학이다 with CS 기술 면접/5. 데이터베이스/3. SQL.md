# SQL

# 데이터 정의 언어(DDL)

- CREATE: 데이터베이스 혹은 데이터베이스 객체 생성
- ALTER: 데이터베이스 객체 갱신(테이블에 필드 및 제약 조건 추가/삭제)
- DROP: 데이터베이스 객체 삭제
- TRUNCATE: 테이블 구조를 유지한 채 모든 레코드 삭제

## CREATE

CREATE 명령어는 데이터베이스, 테이블, 뷰, 인덱스 등까지 데이터베이스에서 관리될 수 있는 다양한 대상을 정의합니다. 

```jsx
CREATE TABLE 테이블 이름(
		필드이름1 필드타입1,
		필드이름2 필드타입2,
		필드이름3 필드타입3 NOT NULL,
		...
)
```

필드 타입 우측에는 특정 필드가 지켜야할 제약 조건을 명시할 수 있습니다. 

- PRIMARY KEY: 특정 필드를 기본 키로 지정
- UNIQUE: 특정 필드가 고유한 값을 갖도록 설정
- FOREIGN KEY: 특정 필드를 외래 키로 지정
- DEFAULT : 기본값 설정
- NULL/NOT NULL: 특정 필드에 NULL값 허용/허용하지 않음

## ALTER

ALTER 명령은 CREATE TABLE문을 통해 생성된 테이블에 새로운 필드를 추가하거나 기존의 필드를 수정/삭제할 수 있고, 제약 조건 또는 새롭게 추가하거나 수정/삭제할 수 있습니다. 

## DROP

DROP은 테이블이나 데이터베이스를 삭제할 수 있는 명령어이다. 

```jsx
DROP TABLE 테이블이름;
DROP DATABASE DB이름;
```

## TRUNCATE

테이블 레코드를 모두 삭제하되, 테이블 자체를 삭제하지 않습니다. 

```jsx
TRUNCATE TABLE 테이블 이름;
```

# 데이터 조작 언어(DML)

- select: 테이블의 레코드 조회
- insert: 테이블의 레코드 삽입
- update: 테이블의 레코드 수정
- delete: 테이블의 레코드 삭제

### GROUP BY

이름처럼 특정 필드를 기준으로 필드를 그룹화하기 위해 사용되며, 앞에서 언급한 연산/집계 함수와 함께 사용되는 경우가 많음

### HAVING

HAVING 절은 GROUP BY 절로 그룹화된 결과에 조건을 적용하기 위해 사용됩니다. 

### ORDER BY

특정 필드를 기준으로 데이터를 정렬할 때 사용됩니다. 오름차순(ASC)으로 정렬이 기본이되, DESC 키워드를 사용해 내림차순으로 정렬할 수 있습니다. 

### LIMIT

조회할 레코드 수를 제한하기 위해 사용됩니다.

# 트랜잭션 제어 언어(TCL)

TCL은 트랜잭션 제어하는데 사용되는 SQL 명령어들입니다. 

- commit: 데이터베이스에 작업 반영
- rollback: 작업 이전의 상태로 되돌림
- savepoint: 롤백 기준점 설정

## 데이터 제어 언어(DCL)

- grant: 사용자에게 권한 부여
- revoke: 사용자로부터 권한 회수
